Security Architecture

Military-grade.
By design.

Every piece of data in every AI Pieces product is protected by the same security stack used in military and intelligence-grade applications.

🔑

AES-256-GCM Encryption

All family data — messages, location, health records, calendar — encrypted using AES-256 in Galois/Counter Mode. AEAD provides both confidentiality and message authentication. Encryption occurs on-device before any data leaves your phone.

crypto.subtle.generateKey({ name: "AES-GCM", length: 256 })
  • 128-bit random IV per encryption operation
  • Authentication tag prevents tampering
  • Zero-knowledge — encrypted ciphertext only stored server-side
🤝

ECDH P-256 Key Exchange

When a new family member joins, their device generates a unique ECDH P-256 key pair. The family AES key is encrypted using the new member's public key via ECDH shared secret derivation — key transmission never occurs in plaintext.

crypto.subtle.generateKey({ name: "ECDH", namedCurve: "P-256" })
  • Private key never leaves the device
  • Public keys stored in Supabase for key exchange only
  • Forward secrecy via per-session derived keys
🏦

Hardware-Backed Key Storage

Private keys stored exclusively in hardware security modules — iOS Secure Enclave and Android StrongBox. Keys are bound to the device and cannot be extracted even if the device is compromised.

securityLevel: Keychain.SECURITY_LEVEL.SECURE_HARDWARE
  • iOS: Secure Enclave (dedicated ARM coprocessor)
  • Android: StrongBox HSM or TEE fallback
  • ACCESSIBLE.WHEN_UNLOCKED_THIS_DEVICE_ONLY
🛡️

Row Level Security (RLS)

Supabase Row Level Security policies enforce that every database query is scoped to the authenticated user's family ID. No cross-family data access is possible at the database layer, even with a valid JWT.

CREATE POLICY family_isolation ON location_updates USING (family_id = auth.uid())
  • PostgreSQL-level enforcement
  • Applies to all tables: messages, locations, health, calendar
  • Audited against SQL injection patterns
🪤

Intrusion Detection — Canary Tokens

Canary token service embeds unique, invisible tracking markers in sensitive data structures. If any data is accessed outside normal application flows, the canary triggers an immediate encrypted alert. Honeypot records monitor for unauthorised enumeration.

CanaryTokenService + IntrusionDetectionService
  • Silent tripwire on sensitive family records
  • Honeypot rows detect automated scraping
  • Encrypted alert to family admin on trigger
🔏

Biometric Authentication

FaceID, TouchID, and Android biometric authentication required before decrypting any family data. Hardware-backed biometric binding via react-native-keychain. Family data cannot be read without biometric verification.

BiometricAuth.ts — ACCESSIBLE.WHEN_UNLOCKED_THIS_DEVICE_ONLY
  • iOS: FaceID / TouchID via Secure Enclave
  • Android: Biometric Manager + StrongBox
  • No plaintext data accessible pre-authentication

Product 02

HealthCompass

Hospital patient-family communication —
encrypted, HIPAA-compliant, staff-friendly.

Built on the FamilyCompass encrypted stack, adapted for clinical environments. Eliminates the single biggest drain on nurse time — repetitive family phone calls — without adding a single minute of admin burden.

Clinical Staff Features

Designed for ICU, maternity, paediatric, and surgical ward nursing staff

📱 Mobile-First, No Computers Required

  • Mobile voice recording — speak a patient update, app converts to structured clinical note via on-device speech-to-text
  • Photo-to-text OCR captures handwritten notes, medication charts, or wound photos from the bedside
  • One-tap status updates — "Stable", "Resting", "In Surgery", "Awake" — broadcast to family instantly without typing
  • Batch update mode — update multiple patient families in under 2 minutes at shift handover
  • Offline mode queues all updates for delivery when connectivity returns

🔗 System Integration

  • REST API integration with Epic, Cerner, and Meditech for automated admission/discharge triggers
  • HL7 FHIR-compatible data structures for patient records and status updates
  • Single Sign-On for hospital staff — integrates with Active Directory / Azure AD
  • HIPAA audit trail — every family notification logged with timestamp, staff ID, and content hash
  • Custom department configuration — ICU, maternity, and paediatric workflows configured independently

👨‍👩‍👧 Family-Facing Features

  • Real-time encrypted push notifications to family members' smartphones
  • Structured update feed replaces phone calls — "11:30am: Settled after procedure, resting comfortably"
  • Controlled visiting window notifications — family informed of visiting hours and room changes
  • Encrypted family messaging channel to nursing station
  • Multi-language family app — critical for international patients

📊 Compliance & Analytics

  • HIPAA-compliant end-to-end encryption — same AES-256-GCM / ECDH stack as FamilyCompass
  • Zero-knowledge architecture — AI Pieces cannot read patient data
  • Department-level dashboards for nursing directors — update frequency and call reduction metrics
  • Automated CQC/JCI inspection readiness reports
  • GDPR-compliant patient consent workflows with digital consent logging

Encrypted family communication
for nursing homes & residential care.

CareCompass brings the privacy and security of FamilyCompass to elderly care — connecting residents' families with care teams without adding burden to already stretched care staff.

Care Staff Features

Designed for nursing homes, assisted living, residential care facilities (20–200+ beds)

📱 Effortless Staff Updates

  • Mobile voice recording and photo OCR — carers capture resident updates from the bedside without a computer
  • Daily wellbeing updates (mood, appetite, activity, social engagement) sent to family via encrypted notification
  • Medication administration logging with timestamp and carer ID for compliance documentation
  • Wound care and health observation photo logging — encrypted, visible only to authorised family and clinical staff
  • Batch update tool for morning rounds — update all resident families during or immediately after rounds

🏠 Resident Safety Features

  • GPS-based wandering detection — encrypted alert to designated carer and family when resident leaves safe zone
  • Fall detection integration via wearable devices (Apple Watch, Garmin, Withings)
  • Medication reminder and non-compliance alert system
  • Emergency contact cascade — encrypted alerts sent to family tiers in defined sequence
  • Activity tracking — meals, physio sessions, and social activities logged and shared with family

👨‍👩‍👧 Family Engagement

  • Daily encrypted update feed — family sees what their relative ate, how they slept, what activities they participated in
  • Encrypted family messaging to care team — replaces phone calls that interrupt staff during care delivery
  • Visiting coordination — families book visits, care team confirms, no phone tag
  • Multi-family access with privacy partitions — different family members see only their relative's data
  • GP appointment notifications — families informed of upcoming medical appointments automatically

📋 Compliance & Inspection Readiness

  • GDPR-compliant from day one — built in Ireland, EU data residency available
  • CQC (UK) / HIQA (Ireland) inspection documentation — automated care communication records with timestamps
  • Consent management — digital family consent for communication and data sharing logged and auditable
  • Staff activity audit trail — every update logged with carer ID and time
  • Data retention policies configurable per facility in line with regulatory requirements
